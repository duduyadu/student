## 3. 작업 순서 근거

```
테스트 → 버그수정 → UI개선 → PDF개선 → 신기능 → 최적화

이유:
- 테스트를 먼저 해야 실제 문제 파악 가능
- 버그 있는 상태에서 디자인 작업 시 이중 수정 발생
- 기능 완성 후 코드 최적화 → 중복 최적화 방지
- 신기능은 기존 기능이 안정된 후 추가
```
