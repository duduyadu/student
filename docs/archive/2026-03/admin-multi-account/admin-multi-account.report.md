### 2-2. 함께 수정된 버그 (5건)

#### BUG-1: 대시보드 Application Error (크리티컬)
- **원인**: `health.checks.map()` 호출 시 `checks` 필드가 undefined (인증 실패 시 `{ error: 'Unauthorized' }` 반환)
- **수정**: `(health.checks ?? []).map()` null guard 추가, `health.checkedAt` 조건부 렌더링
- **파일**: `app/page.tsx`

#### BUG-2: 로그아웃 후 재로그인 불가 (크리티컬)
- **원인**: `lib/supabase.ts`에서 `createClient` (localStorage 기반) 사용 — Next.js middleware는 `@supabase/ssr`의 `createServerClient` (cookie 기반) 사용 → 세션 불일치로 항상 `/login` 리다이렉트
- **수정**: `createBrowserClient` (`@supabase/ssr`)로 교체 → cookie/localStorage 동기화
- **파일**: `lib/supabase.ts`

```typescript
// 수정 전
import { createClient } from '@supabase/supabase-js'
export const supabase = createClient(URL, ANON_KEY)

// 수정 후
import { createBrowserClient } from '@supabase/ssr'
export const supabase = createBrowserClient(URL, ANON_KEY)
```

#### BUG-3: 대시보드 최근 상담 400 Bad Request (2건)
- **원인 1**: `consultations` 쿼리에서 존재하지 않는 `content` 컬럼 선택 → 실제 컬럼은 `summary`
- **원인 2**: `student:students(name_kr)` embedded join이 FK 해석 실패
- **수정**: 컬럼명 `content` → `summary` 수정, embedded join 제거 후 2-step 쿼리 (studentId 목록 → `students` 테이블 별도 조회)
- **파일**: `app/page.tsx`

#### BUG-4: `/api/health` 401 콘솔 에러 (중간)
- **원인**: 대시보드에서 CRON_SECRET 없이 health API 호출 → 항상 401
- **수정**: CRON_SECRET 외 Supabase master JWT도 인증 수단으로 허용 (`app_metadata.role === 'master'` 검증)
- **파일**: `app/api/health/route.ts`, `app/page.tsx` (Bearer 토큰 전달)

#### BUG-5: Recharts `width(-1) height(-1)` 콘솔 경고 (낮음)
- **원인**: `ResponsiveContainer`를 `next/dynamic(ssr:false)` 블록에 포함 → ResizeObserver가 DOM 레이아웃 전 발화 → 음수 크기 계산
- **수정**: `ResponsiveContainer` static import로 변경, `height="100%"` → `height={176}` (고정 px)
- **파일**: `app/page.tsx`

## 4. 아키텍처 인사이트

### Supabase 세션 저장소 불일치 문제
```
createClient (@supabase/supabase-js)
  → localStorage 저장
  → middleware(cookie)가 읽을 수 없음 → 항상 로그인 페이지로 리다이렉트

createBrowserClient (@supabase/ssr)
  → cookie + localStorage 동기화
  → middleware(cookie)가 정상 읽음 → 세션 유지
```

이 패턴은 `@supabase/ssr`를 사용하는 Next.js App Router 프로젝트에서 **반드시** `createBrowserClient`를 사용해야 하는 이유다.

### Critical Thinking First 원칙 채택
이 세션에서 "로그인이 된다 → 안 된다"를 반복한 원인: 에러 증상만 보고 빠른 수정 시도 → 근본 원인(localStorage/cookie mismatch) 확인 지연. CLAUDE.md에 비판적 사고 원칙을 추가하여 재발 방지.
