# 통계/리포트 (Reports & Analytics) — Design

## 1. 개요

학생 현황 통계, 감사 로그, TOPIK 시험 일정을 한 페이지(`/reports`)에서 탭으로 구분하여 제공한다.

- **master**: 통계·감사 로그·TOPIK 일정 3개 탭 모두 접근 가능
- **agency**: 통계 탭만 접근 가능 (자신의 유학원 학생 데이터에 RLS 자동 적용)

---

## 2. 표시 데이터 항목

### 2-1. 통계 탭 (`stats`)

| 항목 | 설명 |
|------|------|
| 상태별 학생 분포 | 활성 학생(`is_active = true`)을 status별로 집계 |
| 월별 등록 추이 | 현재 연도 1~12월 신규 등록 학생 수 (created_at 기준) |
| 유학원별 학생 수 | 유학원명별 활성 학생 수, 내림차순 정렬 |

**학생 상태 목록** (`lib/constants.ts`)
```
['유학전', '어학연수', '대학교', '취업']
```

### 2-2. 감사 로그 탭 (`audit`) — master 전용

| 항목 | 설명 |
|------|------|
| 시간 | KST 기준 월/일 시:분 형식 |
| 액션 | INSERT / UPDATE / DELETE / LOGIN / LOGOUT |
| 테이블 | 변경된 대상 테이블명 |
| 사용자 | user_name (없으면 user_id 앞 8자) |
| 역할 | user_role |
| 상세 | details JSON (최대 60자 truncate) |

표시 건수: 최신 100건 (총 건수 별도 표시)

### 2-3. TOPIK 일정 탭 (`topik`) — master 전용

| 항목 | 설명 |
|------|------|
| 시험 유형 | TOPIK I / TOPIK II |
| 회차 | 제N회 |
| 시험일 | exam_date (YYYY-MM-DD) |
| 지역 | 기본값 "전국" |
| 접수 기간 | reg_start ~ reg_end (선택) |
| D-day | 시험일까지 남은 일수 (과거 시험은 표시 안 함) |

## 3. 차트/시각화 구성

모든 시각화는 외부 차트 라이브러리 없이 Tailwind CSS 바 차트로 구현한다.

### 3-1. 상태별 학생 분포 — 가로 진행 바

```
[유학전  ]  ████████░░░░  12명 (60%)
[어학연수]  ███░░░░░░░░░   5명 (25%)
[대학교  ]  ██░░░░░░░░░░   2명 (10%)
[취업    ]  █░░░░░░░░░░░   1명  (5%)
                            전체 20명
```

- 바 높이: `h-2.5`
- 상태별 색상:

| 상태 | 색상 |
|------|------|
| 유학전 | `bg-slate-400` |
| 어학연수 | `bg-blue-400` |
| 대학교 | `bg-violet-400` |
| 취업 | `bg-emerald-400` |

### 3-2. 월별 등록 추이 — 세로 바 차트

```
 3
 2    █
 1  █ █       █
    1 2 3 ... 12월
```

- 높이 고정 96px 컨테이너, 최대값 기준 비율 계산
- 바 색상: `bg-blue-400`
- 0건인 달은 숫자 표시 생략

### 3-3. 유학원별 학생 수 — 가로 바 차트

```
Trung tâm A  ████████████████  15
Trung tâm B  ████████░░░░░░░░   8
미배정        ██░░░░░░░░░░░░░░   2
```

- 바 색상: `bg-emerald-400`
- 최대값 기준 비율 계산
- `user_id` 미연결(`__unassigned__`) → "미배정" 표시 (i18n key: `unassigned`)

## 4. 데이터 소스 (Supabase 쿼리)

### 4-1. 상태별 분포 (`loadStatusStats`)

```sql
SELECT status
FROM students
WHERE is_active = true
```

클라이언트에서 집계:
- `Map<status, count>` 생성
- `STUDENT_STATUSES` 배열 순서 유지 (`lib/constants.ts`)
- total = 전체 행 수

### 4-2. 월별 등록 추이 (`loadMonthStats`)

```sql
SELECT created_at
FROM students
WHERE is_active = true
  AND created_at >= '{year}-01-01'
  AND created_at <= '{year}-12-31'
```

- 현재 연도 고정
- `created_at.slice(5, 7)` 로 월 추출
- 1~12월 전체 초기화 후 집계 (등록자 없는 달도 0으로 표시)

### 4-3. 유학원별 학생 수 (`loadAgencyStats`)

```sql
SELECT agencies(agency_name_vn, agency_name_kr)
FROM students
WHERE is_active = true
```

- agency가 null이면 `__unassigned__` 처리
- 내림차순 정렬 후 반환

### 4-4. 감사 로그 (`loadAuditLogs`)

```
GET /api/audit?limit=100
Authorization: Bearer {access_token}
```

Response: `{ data: AuditLog[], count: number }`

### 4-5. TOPIK 일정 (`loadTopikList`)

```sql
SELECT *
FROM topik_schedules
ORDER BY exam_date ASC
```

## 5. 역할별 접근 권한

| 기능 | master | agency |
|------|:------:|:------:|
| `/reports` 페이지 접근 | O | O |
| 통계 탭 | O | O (본인 유학원 학생만, RLS 자동 적용) |
| 감사 로그 탭 | O | X (탭 자체 미표시) |
| TOPIK 일정 탭 (조회) | O | X (탭 자체 미표시) |
| TOPIK 일정 추가 | O | X |
| TOPIK 일정 삭제 | O | X |

탭 전환 UI (`tabStats` / `tabAuditLog` / `tabTopikSchedule`)는 `user.role === 'master'` 조건으로 master에게만 렌더링.
agency는 탭 없이 통계 섹션만 표시.

## 6. UI 레이아웃

### 페이지 헤더

```
[통계]                    [통계 | 감사 로그 | TOPIK 일정]  ← master만
                           ^탭 전환 버튼 그룹^
```

### 통계 탭 레이아웃

```
┌──────────────────────────┐  ┌──────────────────────────┐
│ 상태별 학생 분포          │  │ 월별 등록 추이 (올해)     │
│ [수평 바 차트]            │  │ [수직 바 차트]            │
└──────────────────────────┘  └──────────────────────────┘
┌─────────────────────────────────────────────────────────┐
│ 유학원별 학생 수                                         │
│ [수평 바 차트]                                          │
└─────────────────────────────────────────────────────────┘
```

- 상단 2개 카드: `grid grid-cols-1 md:grid-cols-2`
- 하단 유학원별 카드: 전체 너비

### 감사 로그 탭 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│ 감사 로그 (최근 100건 / 총 N건)        [새로고침]        │
│ 시간 | 액션 | 테이블 | 사용자 | 역할 | 상세             │
│ ...                                                      │
└─────────────────────────────────────────────────────────┘
```

액션 뱃지 색상:

| 액션 | 색상 |
|------|------|
| INSERT | `emerald` |
| UPDATE | `blue` |
| DELETE | `red` |
| LOGIN | `violet` |
| 기타 | `slate` |

### TOPIK 일정 탭 레이아웃

```
┌─────────────────────────────────────────────────────────┐
│ TOPIK 일정 관리                        [+ 일정 추가]     │
│                                                          │
│ [추가 폼] (grid 2열, 3열)                               │
│  회차 | 시험일 | 유형 | 접수시작 | 접수종료 | 지역       │
│                                    [저장] [취소]         │
│                                                          │
│ ┌──────────────────────────────────────────── D-45 ──┐  │
│ │ TOPIK I   제91회 — 2026-05-10   전국               │  │
│ │ 접수기간: 2026-03-01 ~ 2026-03-31         [삭제]   │  │
│ └───────────────────────────────────────────────────┘  │
│                                                          │
│ ┌──────────────────────────────────────── (과거 흐림) ┐  │
│ │ TOPIK I   제90회 — 2026-01-18   전국               │  │
│ └───────────────────────────────────────────────────┘  │
└─────────────────────────────────────────────────────────┘
```

D-day 뱃지 색상:
- 7일 이내: `red` (즉시 주의)
- 8~30일 이내: `amber`
- 31일 이상: `blue`
- 과거 시험: D-day 미표시, 카드 `opacity-60`

### TOPIK 일정 추가 폼 입력 필드

| 필드 | 타입 | 필수 | 기본값 |
|------|------|:----:|--------|
| 회차 (`round`) | number | O | — |
| 시험일 (`exam_date`) | date | O | — |
| 시험 유형 (`exam_type`) | select | O | TOPIK I |
| 접수 시작 (`reg_start`) | date | — | — |
| 접수 종료 (`reg_end`) | date | — | — |
| 지역 (`region`) | text | O | 전국 |
